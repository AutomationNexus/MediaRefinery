# Repository Structure

MediaRefinery keeps the root focused on project entry points and community files, while detailed manuals live under `docs/`.

## Root Files

| Path | Purpose |
|------|---------|
| `README.md` | Project overview, quick start, and documentation map. |
| `CHANGELOG.md` | User-visible changes by release. |
| `CONTRIBUTING.md` | Contributor expectations and pull request workflow. |
| `CODE_OF_CONDUCT.md` | Community behavior expectations. |
| `SECURITY.md` | Vulnerability reporting and secure defaults. |
| `SUPPORT.md` | Where to ask questions, report bugs, and request features. |
| `MAINTAINERS.md` | Maintainer responsibilities and release ownership. |
| `LICENSE` | MIT license. |
| `Dockerfile` | Production image build. |
| `pyproject.toml` | Python package metadata and tooling configuration. |

## Source And Tests

| Path | Purpose |
|------|---------|
| `src/mediarefinery/` | Python package. |
| `src/mediarefinery/service/` | FastAPI service, routers, auth, state store, scheduler, and scan runner wiring. |
| `frontend/` | React/Vite dashboard. |
| `tests/` | Python unit and integration tests. |

## Documentation

| Path | Purpose |
|------|---------|
| `docs/getting-started/` | Installation and first deployment. |
| `docs/guides/` | User-facing dashboard workflows. |
| `docs/admin/` | Configuration, operations, and troubleshooting. |
| `docs/reference/` | API and compatibility reference. |
| `docs/security/` | Threat model and security notes. |
| `docs/development/` | Contributor setup and repository structure. |
| `docs/releases/` | Release checklist and audit records. |
| `docs/architecture/` | Architecture decision records. |
| `docs/models/` | Curated model catalog metadata and license texts. |

## Project Automation

| Path | Purpose |
|------|---------|
| `.github/workflows/ci.yml` | Backend tests, linting, type checks, package smoke, and Docker smoke. |
| `.github/workflows/frontend.yml` | Frontend type checks, tests, and build. |
| `.github/workflows/release.yml` | Multi-arch GHCR image publish and GitHub Release creation on tags. |
| `.github/ISSUE_TEMPLATE/` | Public issue templates. |
| `.github/PULL_REQUEST_TEMPLATE.md` | Pull request checklist. |
| `.github/CODEOWNERS` | Optional review routing. |

## Templates

| Path | Purpose |
|------|---------|
| `templates/env.example` | Placeholder note — MediaRefinery has no operator env vars; key file is auto-created at `/data/master.key`. |
| `templates/docker-compose.example.yml` | Example Compose service. |
| `templates/config*.yml` | Advanced internal pipeline examples retained for validation and experiments. |
| `templates/issue-template.md` | Portable issue template mirror. |
| `templates/pr-template.md` | Portable pull request template mirror. |

## Generated Or Local-Only Paths

These paths are ignored by git:

- `.venv/`, caches, and build outputs;
- `src/mediarefinery/web/` generated by `frontend/npm run build`;
- `tmp/`, `build/`, `dist/`, and local data outputs;
- `.env`, `.env.*`, SQLite databases, model weights, logs, and user media.

Do not commit generated dashboard assets, secrets, local databases, thumbnails, frames, originals, or model weights.
