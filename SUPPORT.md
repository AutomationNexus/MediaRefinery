# Support

Use the right channel so maintainers can help without exposing private media or secrets.

## Questions

Open a GitHub Discussion when enabled, or open an issue with the `type:question` label if discussions are not available.

Include:

- MediaRefinery version or commit;
- installation method;
- Immich version;
- whether this is Docker, Compose, or source;
- redacted logs when relevant.

## Bugs

Open a GitHub issue using the bug template.

Include:

- what happened;
- what you expected;
- steps to reproduce;
- relevant environment details;
- redacted logs.

Do not include API keys, passwords, session cookies, full private URLs with tokens, screenshots of private media, database files, thumbnails, originals, or sampled frames.

## Feature Requests

Open a GitHub issue using the feature template. Describe the workflow, why it matters, and any privacy or safety impact.

## Security Reports

Do not open public issues for undisclosed vulnerabilities. Follow [SECURITY.md](SECURITY.md).

## Commercial Support

This repository does not currently provide commercial support contracts.
